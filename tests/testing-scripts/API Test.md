# API Testing

## For validation and regression testing, run the Python test runners from the repository root.

- `python "tests/testing-scripts/full_api_orca_fdm_test_runner.py"`
	- Runs **Orca FDM** full-matrix slicing against all supported files under `tests/testing-files/`.
	- Endpoint: `/orca/slice`
	- Writes report to `tests/testing-scripts/results/full_api_orca_fdm_test_result.md`

- `python "tests/testing-scripts/full_api_prusa_fdm_test_runner.py"`
	- Runs **Prusa FDM** full-matrix slicing against all supported files under `tests/testing-files/`.
	- Endpoint: `/prusa/slice`
	- Writes report to `tests/testing-scripts/results/full_api_prusa_fdm_test_result.md`

- `python "tests/testing-scripts/full_api_prusa_sl1_test_runner.py"`
	- Runs **Prusa SL1 (SLA)** full-matrix slicing against all supported files under `tests/testing-files/`.
	- Endpoint: `/prusa/slice`
	- Writes report to `tests/testing-scripts/results/full_api_prusa_sl1_test_result.md`

- `python "tests/testing-scripts/full_api_test_runner.py"`
	- Suite wrapper that runs all three matrix runners above in sequence.
	- Writes consolidated suite summary to `tests/testing-scripts/results/full_api_test_result.md`.

- `python "tests/testing-scripts/pricing_cycle_test_runner.py"`
	- Runs pricing lifecycle validation for both FDM and SLA in one cycle.
	- Performs create → verify → update → verify → delete → verify using admin API key read from `.env` (with env fallback).

- `python "tests/testing-scripts/admin_output_files_test_runner.py"`
	- Validates protected `GET /admin/output-files` behavior.
	- Checks unauthorized rejection and authorized success response schema (tries `.env` key first, then env key).


- `python "tests/testing-scripts/queue_concurrency_test_runner.py" --count 5 --retry-on-429 3`
	- Runs queue behavior test with `N` concurrent slicing POST requests.
	- Uses `N` supported files from `tests/testing-files/` (or `--file` to force one file).
	- Slicing engine depends on selected endpoint:
	  - `--endpoint /prusa/slice` -> **Prusa**
	  - `--endpoint /orca/slice` -> **Orca**
	- Automatically retries `429` responses (`Retry-After`) so higher `N` still testable under rate limit.
	- Verifies whether completion order follows arrival order (queue semantics).

---

## Test reports are written to `tests/testing-scripts/results/`:

- `admin_output_files_test_result.md`
	- Structured outputs from admin output-files endpoint validation.

- `full_api_orca_fdm_test_result.md`
	- Structured outputs from Orca FDM full-matrix regression runs.

- `full_api_prusa_fdm_test_result.md`
	- Structured outputs from Prusa FDM full-matrix regression runs.

- `full_api_prusa_sl1_test_result.md`
	- Structured outputs from Prusa SL1 full-matrix regression runs.

- `full_api_test_result.md`
	- Consolidated summary generated by the suite wrapper (`full_api_test_runner.py`).

- `pricing_cycle_test_result.md`
	- Structured outputs from pricing lifecycle integration runs.

- `queue_concurrency_test_result.md`
	- Concurrent queue behavior outputs (arrival-order + staggered completion analysis).
